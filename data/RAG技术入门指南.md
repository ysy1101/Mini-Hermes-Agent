# RAG 技术入门指南

## 什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与大语言模型相结合的技术。

简单来说，它的工作方式是：
1. 用户提问
2. 系统从知识库中检索相关文档
3. 将检索到的文档和问题一起交给大模型
4. 大模型基于文档内容生成回答

## RAG 的核心优势

与纯大模型回答相比，RAG 有以下优势：

- **知识时效性**：大模型的训练数据有截止日期，RAG 可以接入最新的文档
- **可溯源**：每个回答都能追溯到具体的文档来源
- **幻觉减少**：基于真实文档生成，大幅降低模型编造内容的概率
- **领域专业化**：可以针对特定领域构建专属知识库

## RAG 的技术栈

构建一个 RAG 系统通常需要：

### 文档处理层
- 文档解析：PyPDF2、python-docx、markdown
- 文本切分：按段落、按语义切分，常见块大小 500-1000 字符

### 向量化层
- Embedding 模型：OpenAI text-embedding-3、BGE、M3E
- 向量数据库：ChromaDB、FAISS、Milvus、Pinecone

### 检索层
- 语义搜索：基于向量相似度
- 重排序：对检索结果二次排序提升精度
- 混合检索：向量检索 + 关键词检索

### 生成层
- 大语言模型：GPT-4、Claude、通义千问等
- Prompt 工程：设计合适的提示词模板

## 常见问题

**Q: RAG 和微调有什么区别？**
A: RAG 是在推理时检索外部知识，模型本身不变；微调是改变模型参数来学习新知识。RAG 更灵活，知识更新成本低；微调适合学习特定风格或格式。

**Q: 向量数据库怎么选？**
A: 小规模（<1万文档）用 ChromaDB 或 FAISS；中等规模用 Milvus 或 Qdrant；大规模用 Pinecone 或 Weaviate。

**Q: 文本切分多大合适？**
A: 一般 500-800 字符，约 1-2 个段落。太小会丢失上下文，太大会降低检索精度。
